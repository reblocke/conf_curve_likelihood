# Data management

This repository currently has no committed study-data directory. If a data workflow is added later, use the directory and provenance policy below.

## Directory policy

- `data/raw/`:
  - immutable inputs
  - do not edit in-place
  - do not commit sensitive or large datasets

- `data/external/`:
  - downloaded artifacts (reference datasets, registries, etc.)
  - every artifact must have a provenance sidecar: `<filename>.source.json`

- `data/processed/`:
  - intermediate datasets generated by code

- `data/derived/`:
  - final analysis-ready datasets

## Provenance sidecars

For any downloaded file `data/external/foo.ext`, add `data/external/foo.ext.source.json` containing:

- source URL / DOI
- retrieval date
- version / tag / commit
- checksum (sha256)
- license
- any manual preprocessing notes

## Immutability

If you discover a problem in raw data:

- do not “fix” the raw file
- write a correction step in the pipeline and document it
- keep the corrected output in `data/processed/` or `data/derived/`
